Questions fréquentes (FAQ)

1. Pourquoi l’analyse de sécurité ne fonctionne pas avec du HTML collé ?

L’analyse de sécurité nécessite d’accéder aux informations du serveur (HTTPS, en‑têtes, cookies…). Ces informations ne sont pas présentes dans le code HTML. Elles ne peuvent être obtenues qu’en analysant une URL.

2. Pourquoi certaines URL ne peuvent pas être analysées ?

Certains sites bloquent les requêtes venant d’autres pages pour des raisons de sécurité. C’est ce qu’on appelle la politique CORS. Dans ce cas, l’analyse de sécurité est impossible, même si le site est accessible dans votre navigateur.

3. Comment analyser un site si l’URL ne fonctionne pas ?

Vous pouvez toujours analyser l’accessibilité du site en :

  1. Ouvrant la page dans votre navigateur
  2. Faisant CTRL + U (ou clic droit → Afficher le code source)
  3. Copiant tout le code HTML
  4. Le collant dans la zone prévue sur la page d’accueil

4. Quelle est la différence entre sécurité et accessibilité ?

La sécurité protège les données et empêche les attaques. L’accessibilité garantit que le site peut être utilisé par tout le monde, y compris les personnes en situation de handicap. Les deux sont essentiels, mais ils analysent des aspects différents.

5. Pourquoi mon site apparaît “non sécurisé” alors qu’il fonctionne ?

Un site peut fonctionner normalement tout en ayant des failles de sécurité. Par exemple, il peut manquer de protections comme HSTS, CSP ou des cookies sécurisés. Cela ne bloque pas l’accès au site, mais augmente les risques pour les utilisateurs.

6. Est‑ce que ce scanner détecte les virus ou les malwares ?

Non. Ce scanner analyse uniquement :

  • Les bonnes pratiques de sécurité web
  • Les bonnes pratiques d’accessibilité

Il ne remplace pas un antivirus ou un outil de détection de malware.

7. Est‑ce que l’analyse est fiable ?

Oui, pour les points analysés. Le scanner se base sur des bonnes pratiques reconnues (HTTPS, CSP, alt, labels…). Cependant, il ne peut pas analyser :

  • Le code JavaScript interne
  • Les bases de données
  • Les failles avancées côté serveur

C’est un outil de sensibilisation, pas un audit complet.
